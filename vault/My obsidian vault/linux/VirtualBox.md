# VirtualBox

## Установка

### Установка VirtualBox

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install virtualbox virtualbox-ext-pack
```

**Fedora:**
```bash
sudo dnf install VirtualBox
```

**Arch Linux:**
```bash
sudo pacman -S virtualbox virtualbox-ext-pack virtualbox-host-modules-arch
```

**Windows/macOS:**
1. Скачать с [официального сайта](https://www.virtualbox.org/wiki/Downloads)
2. Запустить установщик

### Установка Extension Pack

Extension Pack добавляет поддержку USB 2.0/3.0, RDP, шифрование и другие функции.

1. Скачать Extension Pack с [официального сайта](https://www.virtualbox.org/wiki/Downloads)
2. В VirtualBox: **Файл → Настройки → Расширения**
3. Нажать **+** и выбрать скачанный файл
4. Принять лицензию

## Создание виртуальной машины

1. **Создать** → Выбрать имя и тип ОС
2. **Память:** минимум 2 ГБ для Ubuntu (рекомендуется 4 ГБ)
3. **Жёсткий диск:**
   - Создать новый виртуальный диск
   - Тип: **VDI** (VirtualBox Disk Image)
   - Формат: **Динамический** (расширяется по мере заполнения)
   - Размер: минимум 25 ГБ для Ubuntu

**Дополнительные настройки (перед запуском):**
- **Система → Процессоры:** 2+ ядра
- **Дисплей → Видеопамять:** 128 МБ
- **Дисплей → Ускорение:** включить 3D-ускорение

## Установка Ubuntu

### Подготовка

1. Скачать ISO-образ Ubuntu с [официального сайта](https://ubuntu.com/download/desktop)
2. В настройках ВМ: **Носители → Контроллер IDE → Пусто**
3. Справа выбрать иконку диска → **Выбрать файл диска** → указать ISO

### Процесс установки

1. Запустить ВМ
2. Выбрать **Try or Install Ubuntu**
3. Выбрать язык → **Установить Ubuntu**
4. Раскладка клавиатуры → **Russian** (или English)
5. **Обычная установка** → галочка **Download updates while installing**
6. **Стереть диск и установить Ubuntu** (не бойтесь — это только виртуальный диск)
7. Выбрать часовой пояс → **Далее**
8. Создать пользователя и пароль
9. Дождаться завершения → **Перезагрузить**

### Установка Guest Additions

Guest Additions улучшает интеграцию: общий буфер, общие папки, автоматическое изменение разрешения.

**Способ 1 (автоматический):**
1. Запустить ВМ с Ubuntu
2. В меню VirtualBox: **Устройства → Подключить образ диска Guest Additions**
3. В Ubuntu откроется окно автозапуска → **Запустить**
4. Ввести пароль → дождаться установки → перезагрузить ВМ

**Способ 2 (вручную, если автозапуск не сработал):**
```bash
# Обновить пакеты и установить зависимости
sudo apt update
sudo apt install build-essential dkms linux-headers-$(uname -r)

# Смонтировать образ (если не смонтирован автоматически)
sudo mkdir -p /media/cdrom
sudo mount /dev/cdrom /media/cdrom

# Запустить установку
cd /media/cdrom
sudo ./VBoxLinuxAdditions.run

# Перезагрузить
sudo reboot
```

**Проверка установки:**
- Должен работать общий буфер обмена
- Окно ВМ должно автоматически подстраиваться под размер окна
- Работают общие папки

## Настройка сети

### Типы сетевых подключений

| Тип | Описание | Когда использовать |
|-----|----------|-------------------|
| **NAT** | ВМ за NAT хоста, имеет доступ в интернет, но не видна из сети | Изолированный доступ в интернет |
| **NAT Network** | Несколько ВМ в одной NAT-сети, видят друг друга | Тестирование сетей ВМ |
| **Сетевой мост (Bridged)** | ВМ получает IP из той же сети, что и хост | Доступ к ВМ из локальной сети |
| **Внутренняя сеть** | Изолированная сеть только между ВМ | Полная изоляция |
| **Хост-только (Host-only)** | Сеть только между хостом и ВМ | Безопасное тестирование |

### Настройка доступа хост ↔ гость

**Вариант 1: Сетевой мост (рекомендуется)**

ВМ будет в той же сети, что и хост, с собственным IP:

1. Выключить ВМ
2. **Настройки → Сеть**
3. **Тип подключения:** Сетевой мост
4. **Имя:** выбрать физический интерфейс хоста (eth0, wlan0 и т.д.)
5. Запустить ВМ

Проверка:
```bash
# В гостевой Ubuntu
ip addr show

# С хоста пинговать гостя
ping <IP-гостевой-машины>

# С гостя пинговать хоста
ping <IP-хоста>
```

**Вариант 2: NAT + Проброс портов**

Если нужен NAT, но с доступом к ВМ:

1. **Настройки → Сеть → Дополнительно → Проброс портов**
2. Добавить правило:
   - **Имя:** SSH
   - **Протокол:** TCP
   - **IP хоста:** 127.0.0.1
   - **Порт хоста:** 2222
   - **IP гостя:** (оставить пустым)
   - **Порт гостя:** 22

Подключение:
```bash
# С хоста к гостю по SSH
ssh -p 2222 user@localhost
```

**Вариант 3: Два адаптера (мост + NAT)**

Для одновременного доступа в интернет и из локальной сети:

1. **Адаптер 1:** NAT (для интернета)
2. **Адаптер 2:** Сетевой мост (для доступа из сети)

В Ubuntu настроить приоритеты интерфейсов (опционально):
```bash
# Проверить маршруты
ip route show

# Интернет должен идти через NAT-адаптер
```

### Настройка статического IP (Ubuntu 20.04+)

Отредактировать Netplan конфиг:
```bash
sudo nano /etc/netplan/00-installer-config.yaml
```

Пример:
```yaml
network:
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.1.100/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
  version: 2
```

Применить:
```bash
sudo netplan apply
```

## Общие папки

### Настройка

1. Выключить ВМ (или можно на лету)
2. **Настройки → Общие папки**
3. Добавить папку:
   - **Путь к папке:** выбрать на хосте
   - **Имя папки:** `shared` (или любое)
   - **Авто-подключение:** ✓
   - **Создать постоянную:** ✓

### Монтирование в Ubuntu

**Автоматическое монтирование:**
```bash
# Добавить пользователя в группу vboxsf
sudo usermod -aG vboxsf $USER

# Перезайти в сессию или перезагрузить
```

Папка появится в `/media/sf_<имя_папки>` (например, `/media/sf_shared`).

**Ручное монтирование:**
```bash
sudo mkdir -p /mnt/shared
sudo mount -t vboxsf shared /mnt/shared
```

**Автомонтирование при загрузке:**
```bash
# Редактировать fstab
sudo nano /etc/fstab

# Добавить строку
shared  /mnt/shared  vboxsf  defaults,user,rw  0  0
```

**Проблемы с правами:**
```bash
# Дать права на папку
sudo chmod 777 /media/sf_shared
# или добавить пользователя в группу vboxsf (см. выше)
```

## Снимки состояния (Snapshots)

Snapshots позволяют сохранить состояние ВМ и откатиться к нему.

### Создание снимка

1. Выбрать ВМ → **Машина → Сделать снимок состояния** (или Ctrl+T)
2. Ввести имя и описание
3. **OK**

### Управление снимками

**Менеджер снимков:** **Машина → Снимки** (или Ctrl+Shift+S)

- **Восстановить:** откат к состоянию снимка
- **Удалить:** удалить снимок (не влияет на текущее состояние)
- **Сделать снимок текущим:** установить точку возврата

### Рекомендации

- Делать снимки **перед** крупными изменениями
- Не хранить слишком много снимков (занимают место)
- Для долгосрочного резервного копирования использовать **Экспорт** (OVA)

## Полезные горячие клавиши

| Клавиша | Действие |
|---------|----------|
| **Host (правый Ctrl по умолчанию)** | Освободить захват мыши/клавиатуры |
| **Host + F** | Полноэкранный режим |
| **Host + L** | Масштабируемый режим |
| **Host + C** | Режим мини-окна |
| **Host + A** | Закрыть окно ВМ (меню закрытия) |
| **Host + R** | Мягкая перезагрузка |
| **Host + T** | Сделать снимок |
| **Host + Q** | Показать меню ВМ |
| **Host + Home** | Виртуальная машина → Меню |
| **Host + Del** | Ctrl+Alt+Del в гостевой ОС |
| **Host + Backspace** | Ctrl+Alt+Backspace в гостевой ОС |

**Сменить Host-клавишу:** **Файл → Настройки → Ввод → Виртуальная машина**

## Решение частых проблем

### Ошибка: не работают USB-устройства

1. Установить Extension Pack
2. Добавить пользователя в группу:
   ```bash
   sudo usermod -aG vboxusers $USER
   ```
3. Перезайти в сессию
4. В настройках ВМ включить USB 2.0/3.0

### Ошибка: VT-x/AMD-V не доступен

1. Войти в BIOS/UEFI хоста
2. Включить виртуализацию:
   - Intel: **Intel Virtualization Technology** / **VT-x**
   - AMD: **SVM Mode** / **AMD-V**
3. Сохранить и перезагрузить

Проверка:
```bash
# Должно вывести не пустой результат
egrep -c '(vmx|svm)' /proc/cpuinfo
```

### Guest Additions не работает

```bash
# Переустановить
sudo apt remove virtualbox-guest-dkms virtualbox-guest-utils virtualbox-guest-x11
sudo apt install --reinstall virtualbox-guest-dkms virtualbox-guest-utils virtualbox-guest-x11
sudo reboot
```

### Проблемы с сетью

**ВМ не видит сеть:**
1. Проверить, включен сетевой адаптер в настройках ВМ
2. Попробовать другой тип подключения (NAT → Мост)
3. В гостевой ОС:
   ```bash
   # Перезапустить сеть
   sudo systemctl restart NetworkManager
   # или
   sudo dhclient -r && sudo dhclient
   ```

**Нет доступа из хоста к гостю:**
- Проверить тип подключения (должен быть Мост или Host-only)
- Проверить firewall в гостевой ОС:
  ```bash
  sudo ufw status
  sudo ufw allow ssh  # если нужен SSH
  ```

### ВМ занимает много места на диске

**Сжать динамический диск:**

1. В гостевой ОС обнулить свободного места:
   ```bash
   sudo apt install zerofree
   # Загрузиться в recovery mode и выполнить
   zerofree /dev/sda1
   ```

2. На хосте сжать диск:
   ```bash
   VBoxManage modifymedium disk <путь_к_диску.vdi> --compact
   ```

### Экспорт/Импорт ВМ

**Экспорт в OVA:**
- **Файл → Экспорт конфигураций** → выбрать ВМ → сохранить .ova

**Импорт OVA:**
- **Файл → Импорт конфигураций** → выбрать .ova
